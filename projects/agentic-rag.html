01 — Overview

From generic retrieval to targeted, agentic search.

RoleSoftware Engineer
DomainCustomer Support · RAG
FocusRetrieval quality · Evaluation
Timeline2024 — Present

The previous chatbot retrieved from a single broad index — good for breadth, poor for precision. Support queries with similar wording but different intents kept landing on the wrong documents.

I redesigned the retrieval flow as an agentic workflow: classify intent first, route to the right index, rewrite the query for that index, retrieve hybridly, then rerank. Every stage was tuned with offline evaluation on a curated set of representative support queries.

03 — Highlights

What I built.

  • 01

    Agentic retrieval workflow

    Upgraded a production RAG customer-support chatbot from general-purpose retrieval to an agentic workflow with intent detection, index routing, LLM-based query rewriting, hybrid retrieval, and reranking.

  • 02

    Intent detection & index routing

    Designed an intent-detection layer to classify user queries and route them to the most relevant index, replacing broad retrieval with index-specific search and higher-quality rewritten queries.

  • 03

    Ingestion & indexing quality

    Improved document ingestion and indexing with Docling-based PDF processing, table-aware chunk enrichment, and HTML H1/H2/H3 hierarchy detection to preserve document context and improve retrieval precision.

  • 04

    Offline evaluation & tuning

    Tuned retrieval relevance with offline evaluation on representative support queries; improved retrieval accuracy by ~40% and user satisfaction by ~60%.
