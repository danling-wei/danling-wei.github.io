01 — Overview

Multilingual product search, standardized at scale.

RoleSoftware Engineer
DomainData ingestion · Search
FocusETL · LLM translation · ES
Timeline2024 — Present

Four upstream APIs, four schemas, multiple languages — and a single global search index that needed to stay consistent across all of them.

I designed an LLM-assisted Dagster pipeline that ingests, normalizes, translates, and indexes product metadata at scale. The system is idempotent, observable, and self-repairing: backfills, schema evolution, and validation failures are all first-class paths, not afterthoughts.

03 — Highlights

What I built.

  • 01

    Multi-source ingestion at scale

    Designed an LLM-assisted data pipeline ingesting 500K+ product records and 100K+ daily deltas from four heterogeneous APIs into Elasticsearch, standardizing multilingual product metadata for global search.

  • 02

    Dagster workflows, built for production

    Built Dagster-based batch and incremental workflows with idempotent processing, schema evolution, backfill and repair paths, validation checks, failure recovery, and observability.

  • 03

    Elasticsearch indexing redesign

    Redesigned Elasticsearch indexing strategy and data processing flow, improving query performance by 3× and reducing manual operations by ~40%.

  • 04

    LLM-assisted translation

    Automated multilingual product translation with an LLM-assisted pipeline, replacing manual translation workflows and contributing to ~$500K estimated annual translation cost savings.
